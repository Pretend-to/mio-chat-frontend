为什么“缓冲”机制更强？

  • KV Cache 友好：标准窗口每次删除首条消息都会让 Prefix 发生位移，导致模型必须重新计算所有 Token 的 KV Cache。
  • 稳定性：缓冲机制将上下文分为“冻结的摘要”和“活跃缓冲区”。在缓冲区填满之前，前缀完全不变，命中率接近 100%。
  • 长效记忆：即便被挤出窗口的消息，也会被归档进 System Prompt，模型依然记得“我是谁”和“我们刚才聊了什么”。
